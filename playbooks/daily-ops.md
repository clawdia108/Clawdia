# Daily Ops Playbook (v2)

## Morning block (30–45 min)
1. Review overdue + due-today activities
2. Build Top 3 priorities (A/B/C)
3. Flag stale deals (no movement > X days)
4. Confirm every active deal has next activity

## Midday block (15 min)
1. Check progress against Top 3
2. Resolve blockers
3. Re-prioritize remaining day

## Afternoon block (20–30 min)
1. Close loop on completed activities
2. Move deal stages where justified
3. Write PM summary and tomorrow's first 3 actions

## Hygiene rules
- No active deal without next activity
- No ambiguous owner on deals/leads
- Mandatory source/qualification fields on new leads
- Duplicate checks daily

## Output Standards

### Bridge Reports (AM/PM)
- Use `templates/user-report.md` format strictly
- Max 15 řádků celkem
- 3 sekce: **Co je dnes důležité** / **Co jsem udělal** / **Potřebuji od tebe**
- Čeština, neformální ale profesionální
- Čísla místo vágních popisů ("5 dealů" ne "několik dealů")
- Žádné task IDs (`TASK-1001`), agent names (`dealops`), timestamps (`2026-03-03T08:20`)
- Žádné generátor watermarky ("Generated by knowledge_sync.py")
- Priority sekce: jen věci vyžadující akci nebo pozornost uživatele
- Actions sekce: jen věci kde čekáme na lidské rozhodnutí

### Agent Internal Outputs
- Mohou obsahovat task IDs, timestamps, technické detaily
- Pole `output_visibility: "internal"` = neposílat uživateli přímo
- Pole `output_visibility: "user_facing"` = musí projít review gate + simplifikací

### Complexity Classification
- `low`: Formatting, scheduling, status aggregation → economy model
- `medium`: Analysis, drafting, scoring → standard model
- `high`: Strategy, multi-step reasoning, cross-domain synthesis → premium model

### Review Requirements
- Všechny `user_facing` outputy → mandatory review gate
- Všechny CRM write operace → Bridge approval
- Všechny external sends → explicit human approval
- Internal `low` complexity → no review needed
