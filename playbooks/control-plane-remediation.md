# Control-Plane Remediation

## Goal

Turn the workspace from prompt-only orchestration into a small, reliable control plane with:

- one canonical agent registry
- structured task handoffs
- generated dashboards from task state
- explicit validation of config consistency

## Target Architecture

### 1. Canonical registry

- Source of truth: `control-plane/agent-registry.json`
- Consumers:
  - `openclaw.agent-army.json`
  - `workspace/openclaw.model-routing.json`
  - `scripts/setup_agent_army_crons.sh`
  - `scripts/check_control_plane.py`

### 2. Structured task layer

- Active tasks: `tasks/open/*.json`
- Archived tasks: `tasks/done/*.json`
- Canonical shape: `tasks/templates/task.json`
- Use task files for ownership, due dates, blockers, and handoffs

### 3. Derived dashboards

- Generated by `scripts/knowledge_sync.py`
- Outputs:
  - `knowledge/AGENT_INSIGHTS.md`
  - `knowledge/TODAY_SUMMARY.md`
  - `reviews/PENDING_REVIEWS.md`

### 4. Role boundaries

- Specialists write only their owned output files
- Bridge reads cross-agent state and escalates
- Reviewer inspects blocked and `needs_review` tasks
- KnowledgeKeeper summarizes state rather than manually re-reading long logs

## Implementation Order

### Phase 1: Control-plane baseline

1. Align agent ids across registry, routing, cron setup, and folder names
2. Add task schema and directories
3. Add `knowledge_sync.py`
4. Add `check_control_plane.py`

### Phase 2: Operational adoption

1. Move work requests from prose into task JSON files
2. Have specialists update task status instead of only markdown prose
3. Run `knowledge_sync.py` after cron jobs or on a short timer
4. Keep public markdown outputs aggregate and sanitized

### Phase 3: Automation hardening

1. Generate cron commands from the registry instead of editing shell manually
2. Add task validation and schema checks in CI
3. Add safe local-only stores for live CRM and message payloads
4. Split sanitized git-tracked state from private operational state

## Immediate Commands

```bash
python3 scripts/check_control_plane.py
python3 scripts/knowledge_sync.py
```

## Success Criteria

- Agent ids no longer drift
- Open work can be counted from `tasks/open/`
- Knowledge views are generated from state, not hand-written from memory
- Review queue is explicit
- Sensitive operational detail stays out of git-tracked files
